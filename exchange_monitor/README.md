# Exchange Monitor

This project monitors specific wallet addresses on Ethereum blockchain and displays their ETH and USDT balance changes in real-time using candlestick charts.

## Features

- Monitors ETH and USDT balance changes for specified wallet addresses
- Stores data in Supabase
- Real-time candlestick charts with multiple timeframes
- Web interface built with Next.js and deployed on Vercel

## Setup

### Backend (Python)

1. Create a `.env` file in the root directory with your Supabase credentials:
```
SUPABASE_URL=your_supabase_url
SUPABASE_KEY=your_supabase_key
```

2. Install Python dependencies:
```bash
pip install -r requirements.txt
```

3. Create the following tables in your Supabase database:

```sql
-- ETH transactions table
create table eth_transactions (
  id bigint generated by default as identity primary key,
  timestamp timestamp with time zone,
  address text,
  delta numeric,
  tx_hash text,
  block bigint
);

-- USDT transactions table
create table usdt_transactions (
  id bigint generated by default as identity primary key,
  timestamp timestamp with time zone,
  address text,
  from_address text,
  to_address text,
  value numeric,
  tx_hash text,
  block bigint
);
```

4. Run the monitor:
```bash
python monitor.py
```

### Frontend (Next.js)

1. Navigate to the web directory:
```bash
cd web
```

2. Create a `.env.local` file with your Supabase credentials:
```
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

3. Install dependencies:
```bash
npm install
```

4. Run the development server:
```bash
npm run dev
```

5. Build for production:
```bash
npm run build
```

## Deployment

1. Deploy the backend monitor to a server or cloud platform that can run Python scripts continuously.

2. Deploy the frontend to Vercel:
   - Connect your GitHub repository to Vercel
   - Add the environment variables in the Vercel project settings
   - Deploy
